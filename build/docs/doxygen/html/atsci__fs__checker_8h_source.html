00028 /*!
00029  * \brief abstract base class for ATSC field sync checker
00030  *
00031  * Processes input samples one at a time looking for
00032  * an occurence of either the field sync 1 or field sync 2 pattern.
00033  *
00034  * Note that unlike atsci_fs_correlator, this class uses the symbol_num in
00035  * input_tag to avoid having to test each symbol position.
00036  *
00037  * For each sample processed, an output sample and an output tag are produced.
00038  * The output samples are identical to the input samples but are delayed by
00039  * a number of samples given by \p delay().  The output tag associated with
00040  * the the given output sample indicates whether this sample is the beginning
00041  * of one of the field syncs or is an ordinary sample.  The tags are defined in
00042  * atsci_sync_tag.h.
00043  *
00044  * For ease of use, the field sync patterns are defined to begin with the
00045  * first symbol of the 4 symbol data segment sync pattern that immediately
00046  * proceeds the actual PN 511 code.  This makes it easier for downstream code
00047  * to determine the location of data segment syncs merely by counting.  They'll
00048  * occur every 832 samples assuming everything is working.
00049  */
